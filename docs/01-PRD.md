# 主应用 PRD · Reminder

## 产品定位

面向**兴趣社群**的**鼓励型打卡提醒** app。

**目标用户**：健忘的人。设计强调正向反馈而非催促/羞辱。

**核心价值主张**：
- 像微信群一样建立兴趣群组（读书会、健身搭子、晨跑群……）
- 群里共享提醒清单，互相打气
- "拍拍" 替代催促 —— 朋友想到你了，不带压力
- 完成时有小庆祝，跳过日不算输

---

## 核心功能模块

### 1. 提醒（Reminder）

- 可以是**私人**（仅自己可见）或**群组**（群成员共享）
- 共用同一张表，通过 `visibility` 字段隔离
- 支持：标题、描述、截止时间、重复规则（每天 / 每周 / 自定义）
- 完成时可附图、文字备注
- 支持评论、emoji 反应

### 2. 群组（Group）

- 用户可创建群组、邀请他人加入
- 角色：群主（owner）/ 普通成员（member）
- 群主可：编辑群信息、移除成员、解散群、置顶提醒
- 邀请方式：链接 + 二维码 + 通讯录
- 单群最大成员数：50（可在管理后台调整）

### 3. 拍拍（Poke）—— 核心差异化功能

- **设计哲学**：朋友想到你了，不是催你
- 必须关联到一条具体的提醒（用户可在设置里开关"允许不关联"）
- 拍拍内容：选鼓励语气（"差一点点"、"想到你了"、"不急慢慢来"）+ 自定义文字
- **被拍方收到**：友好的全屏推送，看到鼓励语而非命令
- **限额**：对同一人每天最多 3 次（服务端强制，可在管理后台调）
- **不是"催促"**：被拍方界面不显示红色徽章 / 紧迫感

### 4. 认领（Claim）

- 群组提醒可被多人认领（"我帮 ta 做"）
- 替代催促 —— 鼓励代办文化
- 独立 Claims 表，多对多关系

### 5. 连胜（Streak） + 保护卡（Shield Card）

- 用户每完成一天有提醒，连胜 +1
- 漏一天不会断 —— 自动消耗一张保护卡
- 用户可主动"跳过日"（不算失败，不消耗保护卡）
- 保护卡获取：连胜 7 天奖励 1 张
- 最大持有：5 张（可在管理后台调）
- 连胜 7 / 30 天有大庆祝（奖杯 + 分享）

### 6. 今日小赢（Today's Wins）

- 把今天已完成的提醒**显眼展示**
- 主页顶部：「你今天已经搞定 N 件啦 · 连胜 X 天 · 还剩 Y 张保护卡」
- 完成动画：贴纸 + 连胜 +1 + 音效暗示

### 7. 加油榜（Encouragement Leaderboard）

- 群组维度，正向排序（不是赖账榜）
- 显示金银铜「冠亚季」，按本周完成数排序
- 落后成员**不显示催促按钮**，改为「想搭把手」（蓝色，引导认领）

### 8. 通知

- in-app 通知：拍拍、认领、完成、邀请、加油榜更新
- Web Push：拍拍、提醒到期、连胜达成（需用户授权）
- iOS：仅在 PWA 安装到主屏后才能收 Push（v1 接受这个限制）

### 9. 标签 / 分类

- 用户可对私人提醒打标签（学习、健身、生活……）
- 标签有图标和颜色
- 可在标签页查看该标签下所有提醒

### 10. 全局搜索

- 跨提醒、群组、评论、用户搜索
- 搜索无结果时引导："要不创建一条新提醒？"

## 用户旅程（关键流程）

### 新用户首次打开

1. Onboarding 页（极简，一屏说清"鼓励而非催促"）
2. 邮箱注册（验证邮件）
3. 引导创建第一条私人提醒，或加入第一个群
4. 提示授权通知（不强制）
5. 提示"添加到主屏"（PWA 安装，为 iOS Push 准备）

### 群组协作流

1. 用户 A 创建群 → 邀请用户 B（链接/QR）
2. 用户 B 通过链接 → 注册（如未注册）→ 自动加入群
3. A 在群里建提醒 → B 收到实时通知
4. B 完成或认领 → A 实时看到状态变化
5. 完成时所有群成员的"加油榜"实时更新

### 拍拍流

1. 用户 A 在群提醒详情页 → 点"拍拍"
2. 选目标人 + 鼓励语气 + 自定义消息
3. 服务端校验：今天对此人是否还有名额（≤3）
4. 推送给被拍方 → 实时 + Web Push
5. 被拍方收到全屏页面（友好、不震惊）

## PRD 决策记录

| 决策项 | 结论 | 备注 |
|---|---|---|
| 平台 | 响应式 web app | 移动端优先，桌面端 v2 |
| 注册方式 | 邮箱 + 密码 | 后期可加 OAuth |
| Reminder 表 | 共享表，`group_id nullable` + `visibility` 字段隔离 | 需严格鉴权 |
| 认领 | 独立 Claims 表，多人可认领 | |
| 拍拍 | 必须关联 reminder_id，可开关 | 服务端限额 3/天/人 |
| 群角色 | owner / member 两级 | |
| 退群数据 | 创建者退群，其提醒**保留**，显示"已退出成员" | |
| 实时 | Socket.io，user/group 双 room | |
| Push | Web Push + Service Worker，iOS 需 PWA | |
| 连胜时区 | 用户注册时自动取，存到 Users 表 | |

## 视觉设计

**风格**：手绘 sketch（精细化版本）。设计稿已完成 **25 个屏**：

**核心 9 屏**：
1. 今天（含今日小赢 + 拍拍提醒）
2. 私人清单
3. 群组列表
4. 群组详情（含加油榜）
5. 创建提醒（底部 sheet）
6. 邀请加入
7. 拍拍流程
8. 提醒详情
9. 个人主页 + 通知

**16 个二级界面**：
- 提醒动作：完成庆祝、跳过日 sheet、改约 sheet、长按菜单
- 拍拍接收：被拍全屏页
- 群组流：建群 sheet、群设置、@ 选人、群邀请陆设页
- 发现：全局搜索、历史小赢、标签管理
- 设置：通知偏好、保护卡确认、7 天大庆祝
- 空状态：首次打开、今天没事、搜索无结果

**字体**：Caveat（标题手写）+ Kalam（正文手写）+ JetBrains Mono（chrome）
**配色**：暖纸 #faf7f1 + 墨黑 + 戳红 + 认领蓝 + 完成绿
**图标**：手绘风 SVG（30+ 图标），不用 emoji（除 user-generated 内容）
